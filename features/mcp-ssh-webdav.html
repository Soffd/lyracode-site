MCP · FTP · SSH · WebDAV ​

Lyra Code 通过多种协议集成远程服务，所有连接均可通过统一的自然语言界面完成配置。

MCP（模型上下文协议） ​

Lyra Code 既可作为 MCP 客户端连接远程服务器，也可作为 MCP 服务端将自身 Agent 工具暴露给局域网客户端。

客户端模式 ​

连接远程 MCP 服务器，扩展 Agent 的能力边界：

  • Streamable HTTP 传输
  • SSE（Server-Sent Events） 传输
  • 随时添加、配置、启用和禁用服务器
  • 每个 MCP 工具以 mcp_ 前缀标识，清晰可辨
  • 所有 MCP 工具调用需用户确认后才能执行

服务端模式（v2.6 新增） ​

Lyra Code 可以作为本地 MCP 服务端运行，将内置 Agent 工具暴露给局域网内其他 MCP 客户端（如 RikkaHub、Operit、Codex、Claude Code、OpenCode 等）：

  • 手机与 PC 协同调用，扩展桌面 AI 工具的移动端能力
  • 工具默认对所有连接的客户端可用，由客户端自行管理调用确认
  • 支持 Streamable HTTP 传输

使用场景 ​

  • 接入专业 AI 工具和数据源
  • 通过 MCP 封装集成外部 API
  • 将手机文件操作、命令执行等能力供给桌面 AI 工具
  • 无需修改应用即可扩展 Lyra Code 的原生能力

客户端模式确认

客户端模式下，MCP 服务器的工具与原生工具并列显示，所有工具调用均需要用户确认后才能执行。

FTP / FTPS / SFTP（v2.6 新增） ​

直接管理远程文件传输服务器的文件，无需离开 AI 对话：

操作说明
file_transfer_list列出远程目录
file_transfer_search按文件名搜索文件
file_transfer_download_to_workspace下载到工作区
file_transfer_upload_from_workspace上传到远程服务器

支持的协议 ​

  • FTP — 标准文件传输协议（明文）
  • FTPS — 隐式/显式 TLS 加密的 FTP
  • SFTP — SSH 文件传输协议

安全提示 ​

  • FTP 是明文协议，涉及敏感文件时建议使用 SFTP 或 FTPS
  • 上传或下载前 Agent 会请求用户确认
  • 服务器配置中可设置被动模式和编码

SSH ​

安全连接远程服务器，进行运维管理和自动化操作：

支持的目标 ​

  • Linux 服务器
  • Windows 服务器
  • Git 服务器

认证方式 ​

  • 密码登录
  • 私钥认证（可选口令短语）

功能特性 ​

  • 远程执行命令并获取 stdout/stderr
  • 每次会话可指定工作目录
  • 为交互式命令提供输入行（如 Y/N 提示）
  • 可配置超时

安全机制 ​

  • 每条 SSH 命令都需要用户明确批准
  • 执行前完整预览命令
  • 使用 list_ssh_servers 查看已配置连接
  • 安装或修改软件前务必先检查系统资源

WebDAV ​

通过 WebDAV 访问和管理远程存储：

功能 ​

操作说明
webdav_list使用 PROPFIND 浏览目录
webdav_search按文件名或路径搜索
webdav_download下载文件到工作区
webdav_upload上传工作区文件到服务器

云备份 ​

  • 直接导出备份到 WebDAV
  • 默认路径：/LyraCode/lyra_backup_latest.zip
  • 导入时自动检测最新备份
  • 支持完整和选择性备份（方案、对话、配置、密钥）

自然语言配置管理 ​

所有 MCP、FTP、SSH、WebDAV、技能包和 Agent 工具配置都可以通过自然语言管理：

"在 https://my-mcp.example.com 添加一个 Streamable HTTP 的 MCP 服务器"
"配置 SSH 连接到我的 VPS，IP 是 192.168.1.100，使用密钥认证"
"添加一个 FTP 连接到我的 NAS，使用 SFTP 协议"
"设置 WebDAV 备份到我的 NextCloud 服务器"
"启用设备硬件检测工具"

你只需描述需求，Agent 会处理配置细节。

安全提示

HTTP API、MCP、FTP 和 WebDAV 端点若非 HTTPS 或受信网络保护，则不安全。生产环境请务必使用加密连接。
